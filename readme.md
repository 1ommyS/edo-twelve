## Глоссарий:

- Функциональные требования отвечают на вопрос ЧТО ДЕЛАЕТ СИСТЕМА
- Нефункциональные требования отвечаеют на вопрос КАК СИСТЕМА РЕАЛИЗУЕТ ФТ и КАКИЕ ОГРАНИЧЕНИЯ НА БЕЗОПАСНОСТЬ/СКОРОСТЬ
  РАБОТЫ/СРОК ХРАНЕНИЯ ИНФОРМАЦИИ

# Функциональные требования

- У пользователя должна быть возможность просматривать просматривать все возможные операции с файлами: создание,
  просмотр, редактирвоание, удаление (create, read, update, delete -> CRUD)
- Пользователь должен иметь возможность поставить пароль на файл
- Доступ к файлу есть только у того, кто знает пароль
- Система должна хранить время создания и время последнего редактирование файла

# Нефункциональные требования

- Пароли к файлу не должны храниться в открытом виде
- Просмотр пароля через средства операционной системы должен быть запрещен
- По файлам должна собираться статистика
- Система после перезагрузки должна сразу знать все о ранее созданных файлах
